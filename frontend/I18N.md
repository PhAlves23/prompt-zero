# Guia de Internacionalização - PromptZero

## Idiomas Suportados

- 🇧🇷 **pt-BR** (Português - Brasil) - idioma padrão
- 🇺🇸 **en-US** (English - United States)
- 🇪🇸 **es-ES** (Español - España)

## Estrutura

```
frontend/
├── app/
│   ├── [lang]/                    # Rotas internacionalizadas
│   │   ├── dictionaries/          # Arquivos de tradução
│   │   │   ├── pt-BR.json
│   │   │   ├── en-US.json
│   │   │   └── es-ES.json
│   │   ├── dictionaries.ts        # Helper de carregamento
│   │   ├── layout.tsx             # Layout com suporte a [lang]
│   │   └── page.tsx               # Página com traduções
│   └── globals.css
├── middleware.ts                  # Detecção automática de idioma
├── lib/i18n.ts                    # Configuração e constantes
└── components/
    └── language-switcher.tsx      # Componente de troca de idioma
```

## Como Usar

### 1. Em Server Components (Recomendado)

```tsx
import { getDictionary, hasLocale, type Locale } from "./dictionaries";
import { notFound } from "next/navigation";

type PageProps = {
  params: Promise<{ lang: string }>;
};

export default async function MyPage({ params }: PageProps) {
  const { lang } = await params;

  if (!hasLocale(lang)) {
    notFound();
  }

  const dict = await getDictionary(lang as Locale);

  return (
    <div>
      <h1>{dict.prompts.title}</h1>
      <button>{dict.common.save}</button>
      <p>{dict.prompts.empty}</p>
    </div>
  );
}
```

### 2. Em Client Components

Passe o dicionário como prop do Server Component pai:

```tsx
// page.tsx (Server Component)
import { ClientComponent } from "./client-component";

export default async function Page({ params }: PageProps) {
  const { lang } = await params;
  const dict = await getDictionary(lang as Locale);
  
  return <ClientComponent dict={dict} lang={lang} />;
}

// client-component.tsx
'use client'

import { type Dictionary } from "./dictionaries";

export function ClientComponent({ 
  dict, 
  lang 
}: { 
  dict: Dictionary; 
  lang: string;
}) {
  return (
    <div>
      <button onClick={() => alert(dict.common.success)}>
        {dict.common.save}
      </button>
    </div>
  );
}
```

### 3. Componente de Troca de Idioma

```tsx
import { LanguageSwitcher } from "@/components/language-switcher";

export default async function Page({ params }: PageProps) {
  const { lang } = await params;
  
  return (
    <div>
      <LanguageSwitcher currentLang={lang as Locale} />
    </div>
  );
}
```

### 4. Metadata Internacionalizada

O `layout.tsx` já está configurado para gerar metadata dinâmica:

```tsx
export async function generateMetadata({ params }: { params: Promise<{ lang: string }> }) {
  const { lang } = await params;
  const dict = await getDictionary(lang as Locale);
  
  return {
    title: dict.meta.title,
    description: dict.meta.description,
  };
}
```

## Rotas e URLs

### Detecção Automática

Quando o usuário acessa `/`, o middleware detecta o idioma do navegador e redireciona:

- Navegador em português → `/pt-BR`
- Navegador em inglês → `/en-US`
- Navegador em espanhol → `/es-ES`

### URLs Diretas

Você pode acessar diretamente qualquer idioma:

- `https://promptzero.dev/pt-BR`
- `https://promptzero.dev/en-US`
- `https://promptzero.dev/es-ES`

### Trocar Idioma

O componente `LanguageSwitcher` muda a URL preservando o resto do path:

- `/pt-BR/dashboard` → `/en-US/dashboard`
- `/es-ES/prompts/123` → `/pt-BR/prompts/123`

## Dicionário Atual

### Grupos disponíveis:

- `common` - Ações comuns (salvar, cancelar, deletar, etc)
- `nav` - Navegação (dashboard, prompts, settings, etc)
- `prompts` - Tela de prompts
- `auth` - Autenticação
- `settings` - Configurações
- `meta` - Metadata (título, descrição, tagline)

### Exemplo completo:

```tsx
import { getDictionary, hasLocale, type Locale } from "./dictionaries";
import { notFound } from "next/navigation";
import { LanguageSwitcher } from "@/components/language-switcher";

export default async function Dashboard({ params }: PageProps) {
  const { lang } = await params;

  if (!hasLocale(lang)) {
    notFound();
  }

  const dict = await getDictionary(lang as Locale);

  return (
    <div className="p-8">
      <div className="flex justify-between items-center mb-8">
        <h1 className="font-heading text-4xl font-bold">
          {dict.nav.dashboard}
        </h1>
        <LanguageSwitcher currentLang={lang as Locale} />
      </div>

      <div className="bg-surface border border-border rounded-2xl p-6">
        <h2 className="font-heading text-xl font-bold mb-4">
          {dict.prompts.title}
        </h2>
        <p className="text-muted-foreground">
          {dict.prompts.empty}
        </p>
        <button className="mt-4 px-6 py-2 bg-pz-lime text-pz-black rounded-lg font-semibold">
          {dict.prompts.create}
        </button>
      </div>
    </div>
  );
}
```

## Boas Práticas

1. **Sempre valide o locale** com `hasLocale()` antes de usar
2. **Use Server Components** sempre que possível (melhor performance)
3. **Mantenha as chaves organizadas** por contexto/grupo
4. **Traduza tudo**, incluindo:
   - Textos visíveis
   - Placeholders
   - Mensagens de erro
   - Metadata (title, description)
   - Alt texts de imagens
5. **Teste em todos os idiomas** antes de fazer deploy

## Adicionar Novo Idioma

Para adicionar um novo idioma (ex: francês):

1. Crie `app/[lang]/dictionaries/fr-FR.json`
2. Adicione em `dictionaries.ts`:
   ```ts
   const dictionaries = {
     'pt-BR': () => import('./dictionaries/pt-BR.json').then((module) => module.default),
     'en-US': () => import('./dictionaries/en-US.json').then((module) => module.default),
     'es-ES': () => import('./dictionaries/es-ES.json').then((module) => module.default),
     'fr-FR': () => import('./dictionaries/fr-FR.json').then((module) => module.default), // novo
   }
   ```
3. Adicione em `middleware.ts`:
   ```ts
   const locales = ['pt-BR', 'en-US', 'es-ES', 'fr-FR']
   ```
4. Adicione em `lib/i18n.ts`:
   ```ts
   export const localeNames: Record<Locale, string> = {
     'pt-BR': 'Português',
     'en-US': 'English',
     'es-ES': 'Español',
     'fr-FR': 'Français', // novo
   };
   ```
5. Adicione em `layout.tsx` no `generateStaticParams()`

## Troubleshooting

### Erro: "locale is not assignable to type Locale"

Use type assertion: `lang as Locale` após validar com `hasLocale(lang)`

### Middleware não está redirecionando

Verifique se o matcher no `middleware.ts` não está bloqueando sua rota.

### Traduções não aparecem

1. Verifique se o arquivo JSON está bem formatado
2. Confirme que a chave existe em todos os idiomas
3. Verifique se está usando `await getDictionary(lang)`
